# AI2 PHP SDK 目录结构

```text
api-sdk/php/
  composer.json
  README.md
  CHANGELOG.md
  docs/
    01_架构设计.md
    02_目录结构.md
    03_API规划.md
  examples/
  tests/
    Unit/
    Integration/
  src/
    functions.php
    Config/
    Client/
    Resource/
    Context/
    Result/
    Exception/
    Contract/
    Support/
    Internal/
```

## src/functions.php

只放两个公开入口函数：

- `merchant()`
- `system()`

## src/Config

存放配置对象：

- `MerchantConfig`
- `SystemConfig`

作用：

- 校验基础配置
- 将数组配置标准化
- 避免把散乱数组在全 SDK 四处透传

## src/Client

存放客户端入口：

- `MerchantClient`
- `SystemClient`

作用：

- 提供资源入口
- 维护当前配置与调度器
- 暴露上下文构造方法，如 `session($sessionId)`

## src/Resource

存放集合级资源，例如：

- `Merchant\Sessions`
- `Merchant\Jobs`
- `Merchant\Stats`
- `System\Merchants`
- `System\Models`

作用：

- 负责列表、创建、集合级查询
- 负责单 ID 可直接访问但不适合强制绑定上下文的资源，例如 `messages()->get($messageId)`

## src/Context

存放单实例上下文，例如：

- `Merchant\SessionContext`
- `Merchant\SessionMessages`
- `System\SessionContext`
- `System\SessionMessages`

作用：

- 固定某个 `session_id`
- 将会话级相关操作聚合在一起

## src/Result

存放结果对象。

当前先放：

- `Common\ApiResult`

后续再逐步演化为更细的：

- `Merchant\SessionResult`
- `System\MerchantResult`

## src/Exception

存放统一异常：

- `Ai2Exception`
- `ApiException`
- `TransportException`
- `InvalidConfigException`

## src/Contract

存放扩展点接口：

- `HttpClientInterface`

后续如有需要再补：

- `LoggerInterface`
- `RequestIdGeneratorInterface`

## src/Support

存放轻量辅助工具：

- `RequestId`
- `Nonce`

## src/Internal

内部实现层，不作为公开兼容承诺：

- `Internal\Http\Dispatcher`
- `Internal\Http\ResponseDecoder`
- `Internal\Http\Transport\GuzzleHttpClient`
- `Internal\Auth\MerchantSigner`
- `Internal\Auth\HeaderBuilder`

这个目录是未来主要演进区。
