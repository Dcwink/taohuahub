# 服务端联动更新清单

## 1. 文档目的

本文档用于约束 `taohuax_ai` 服务端优化过程中，`api-sdk/php` 需要同步更新的内容，避免服务端协议变化后 SDK 文档与实现滞后。

联动原则：

- 服务端每新增或调整一个对外可见接口，都要检查本清单
- 服务端每调整一个对外字段语义，都要检查本清单
- 服务端每冻结一个事件契约，都要同步更新 SDK 文档

---

## 2. 当前联动顺序

优先按以下顺序同步：

1. 更新 SDK 文档
2. 更新 SDK README 调用说明
3. 更新 SDK CHANGELOG
4. 如 SDK 代码已实现对应资源，再更新代码

---

## 3. 当前待同步节点

## A1 运行状态与基础观测能力

若服务端新增以下接口，需同步：

- `system->ops()->queueStatus()`
- `system->ops()->workers()`
- `system->ops()->providerHealth()`
- `system->ops()->failedJobs()`
- `system->ops()->trace($requestId | $sessionId | $jobId)`

需要更新：

- `api-sdk/php/docs/03_API规划.md`
- 后续新增 `ops` 资源设计说明

## A4 SSE / result / async 契约冻结

若服务端明确以下内容，需同步：

- `response_mode=result|stream|async` 语义
- `done/error` 终态字段
- SSE 事件顺序

需要更新：

- `api-sdk/php/README.md`
- `api-sdk/php/docs/03_API规划.md`
- `api-sdk/php/CHANGELOG.md`

## A5 商户鉴权 nonce 重放防护

若服务端对签名规则、时间窗或 nonce 增加约束，需同步：

- Merchant 鉴权说明
- 请求 header 规范
- 错误码说明

需要更新：

- `api-sdk/php/README.md`
- 鉴权与签名实现说明文档

## B1 配置体系分层

若服务端新增配置说明、部署方式或环境变量要求，需同步：

- SDK 示例中的 base URL / header / 环境说明
- SDK README 中对服务端前置要求的说明

## B3 调度层字段变化

若服务端对下列返回字段进行新增或调整，需同步：

- job 状态字段
- provider / target 摘要字段
- usage / billing 相关字段
- queue / retry / cooldown 字段

---

## 4. 当前建议补充的 SDK 资源规划

根据服务端优化方案，后续可增加但暂不强行实现的资源：

- `system->ops()`
- `trace()` 查询能力
- `providerHealth()` 查询能力
- `queueStatus()` 查询能力

说明：

- 在服务端接口未稳定前，SDK 只先补文档设计，不急于写死代码实现
