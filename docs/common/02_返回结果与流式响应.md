# 返回结果与流式响应

## ApiResult

所有公开接口统一返回 `TaohuaHub\Ai2\Result\Common\ApiResult`。

## 常用方法

- `isSuccess(): bool`
- `isFailed(): bool`
- `ret(): int`
- `msg(): string`
- `data(): array`
- `errcode(): string`
- `errmsg(): string`
- `raw(): array`
- `toArray(): array`

## 成功响应

服务端返回：

```json
{
  "ret": 1,
  "msg": "success",
  "data": {}
}
```

SDK 行为：

- `isSuccess() === true`
- `data()` 返回业务 `data`

## 失败响应

服务端返回：

```json
{
  "ret": 0,
  "msg": "参数错误",
  "data": {
    "errcode": "INVALID_PARAMS",
    "errmsg": "缺少 s_session_id 参数"
  }
}
```

SDK 行为：

- `isFailed() === true`
- `errcode()` 和 `errmsg()` 从 `data.errcode` / `data.errmsg` 读取

## `system->messages()->send()` 的流式模式

当请求体传入：

```php
['response_mode' => 'stream']
```

服务端会返回 `text/event-stream`。SDK 会等待流结束，再把 SSE 事件解析进 `ApiResult::data()`：

- `response_mode`
- `content_type`
- `events`
- `terminal_event`
- `body`

终态规则：

- 终态事件是 `done` 时，结果记为成功
- 终态事件是 `error` 时，结果记为失败

## 注意

- merchant 侧不直接消费 SSE 长连接，使用 `sseSign()` + `resolveUrl()`
- `system->messages()->send(['response_mode' => 'async'])` 和 `result` 仍按普通 JSON 返回
