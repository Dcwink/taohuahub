# System AI 消息接口

## 根资源

类：`TaohuaHub\Ai2\Resource\System\Messages`

### `getList(array $query = []): ApiResult`

- Endpoint: `GET /admin/system/v1/ai/messages`
- 常用查询参数：
  - `page`
  - `pagesize`
  - `s_session_id`
  - `s_include_archive`
  - `s_request_id`
  - `s_message_role`
  - `s_content_type`
  - `s_keyword`

说明：

- 服务端要求消息列表必须提供 `s_session_id`

### `get(string $messageId, bool $includeArchive = false): ApiResult`

- Endpoint: `GET /admin/system/v1/ai/messages/detail`

### `send(array $payload): ApiResult`

- Endpoint: `POST /admin/system/v1/ai/messages/send`
- 主要参数：
  - `session_id`
  - `request_id`
  - `sender_actor_type`
  - `sender_actor_id`
  - `sender_actor_name`
  - `response_mode`
  - `content_type`
  - `content`
  - `attachments`

`response_mode` 取值：

- `result`
- `async`
- `stream`

## 会话消息上下文

类：`TaohuaHub\Ai2\Context\System\SessionMessages`

### `getList(array $query = []): ApiResult`

- Endpoint: `GET /admin/system/v1/ai/messages`
- SDK 会强制注入当前会话的 `s_session_id`

### `get(string $messageId, bool $includeArchive = false): ApiResult`

- Endpoint: `GET /admin/system/v1/ai/messages/detail`

### `send(array $payload): ApiResult`

- Endpoint: `POST /admin/system/v1/ai/messages/send`
- SDK 会强制注入当前会话的 `session_id`

## 流式模式说明

当 `response_mode = stream` 时：

- 服务端返回 `text/event-stream`
- SDK 等待流结束
- 返回 `ApiResult`
- SSE 事件位于 `data()['events']`
- 终态事件位于 `data()['terminal_event']`

## 示例

```php
<?php

$result = $system->session('20260411S0A1B2C3D4E')->messages()->send([
    'request_id' => 'req_001',
    'sender_actor_type' => 'user',
    'sender_actor_id' => '10001',
    'response_mode' => 'stream',
    'content' => 'hello',
]);
```
